# 仓库导览

## 先读什么

1. 先读根目录 `README.md`，看仓库整体定位。
2. 再读 `openspec根据项目个性化方法论.md`，看方法论主线。
3. 再读 `openspec 资产迁移/README.md`，看迁移包怎么用。
4. 再读 `../skills/README.md` 和 `../skills/openspec-propose/SKILL.md`，看 skill 角色与 propose 主迁移源资产。
5. 再读 `openspec 资产迁移/QUALITY-GATE.md` 和 `用户说明.md`，看校验与归档规则。
6. 再读 `openspec 资产迁移/schemas/project-schema/README.md` 与 `config.yaml.example`，看 schema 和项目上下文骨架。
7. 再读 `../skills/openspec-apply-change/SKILL.md` 和 `../skills/test-driven-development/SKILL.md`，看默认实施流程。
8. 最后读 `../en/OpenSpec-vs-This-Workflow.md`，看和原版 OpenSpec 的差异。

## 如果你是第一次访问

- 先看 `README.md`
- 再看 `../en/OpenSpec-vs-This-Workflow.md`
- 然后看方法论主文档

## 如果你想复用这套方法

- 先看 `openspec 资产迁移/README.md`
- 再看 `../skills/README.md`
- 再用 `../skills/openspec-propose/SKILL.md` 覆盖目标项目原有的 `propose` skill
- 再看 `config.yaml.example`、`schemas/project-schema/` 和三个门禁文档
- 再接入 `../skills/openspec-apply-change/SKILL.md`
- 然后检查目标项目是否已有 `../skills/test-driven-development/SKILL.md` 对应 skill，缺失就补齐

## 如果你想修改这套方法

- 先改主方法论
- 再改迁移包说明
- 再改 `skills/` 目录下的实现
- 再改 schema 骨架
- 再统一门禁文档
- 最后同步设计说明文档，避免入口规则和 skill 设计说明脱节
